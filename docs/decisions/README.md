# 意思決定記録

目的: 大きな構造変更や方針転換について、何が起き、なぜ変え、何が影響を受けたかを残す。

## ADR を書くタイミング
次のような場合に ADR を書きます。
- リポジトリ構造や作業フローの大きな方向が変わったとき
- 実装や運用の結果、以前の前提が破綻したとき
- 基盤提案や運用ルールに対して、履歴として残すべき修正が入るとき

## 現時点の正本との関係
- 現時点の正本は product / traceability / spec 側に置きます
- ADR は、意思決定イベントとその影響を残します

## 命名規則
- ADR: `adr-YYYYMMDD-<slug>.md`
