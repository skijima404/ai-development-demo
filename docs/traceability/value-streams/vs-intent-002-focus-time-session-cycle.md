## 目的
割り込みの多い業務の中で、ユーザーが短時間でフォーカスタイムを開始し、集中の区切りを認識し、次の行動へ移れるまでの価値の流れを記述する。

## 開始条件
- ユーザーが集中したい作業をこれから始めようとしている
- 作業は設計、レビュー、文書作成、分析、仕様検討など、思考負荷が高い
- ユーザーは複雑な設定ではなく、すぐ始められる道具を求めている

## 価値の流れ
1. 作業前の切り替え:
   ユーザーが「これから集中する」と決める。
2. 時間設定の確認:
   ユーザーが既定の 25 分を使うか、必要に応じて時間を変更する。
3. フォーカスタイム開始:
   ユーザーがタイマーを開始し、いま集中時間であることを認識する。
4. 集中継続:
   ユーザーが残り時間を確認しながら作業を続ける。
5. 終了の認識:
   フォーカスタイムの終了が分かり、区切りがついたことを理解する。
6. 次の行動への移行:
   ユーザーが休憩、通知確認、または次のフォーカスタイム開始へ移る。

## 段階ごとの価値
- 作業前の切り替え:
  - 仕事のモードを意図的に切り替えられる
- 時間設定の確認:
  - 自分の作業に合った長さで無理なく始められる
- フォーカスタイム開始:
  - いま集中すべき時間であることが明確になる
- 集中継続:
  - 感覚任せではなく、時間の枠の中で作業できる
- 終了の認識:
  - 集中の一区切りを迷わず判断できる
- 次の行動への移行:
  - 休憩や通知確認へ自然に切り替えられる

## 重要なプロダクト仮説
- フォーカスタイム開始までの操作が短いほど、日常業務へ取り込みやすい
- 既定値 25 分と変更可能な時間設定の両立が、直感性と柔軟性を両立する
- 実行履歴などを持たない軽量構成でも、最初の価値は十分に伝わる

## この価値の流れの失敗モード
- 開始までの操作が多く、集中前に面倒になる
- 時間設定が分かりにくく、既定値と変更方法の意図が伝わらない
- いま何の状態なのかが分からず、集中時間として機能しない
- 終了が分かりにくく、休憩や通知確認への切り替えが曖昧になる
- MVP の対象外を入れ込みすぎて、直感的な体験が薄れる
