## UX 意図
ユーザーが迷わずフォーカスタイマーを開始し、現在の状態と残り時間を一目で理解できる、単一画面の軽量な体験を作る。

## 利用フロー
1. ユーザーが画面を開く -> 既定値 25 分を確認する -> 必要なら時間を変更する -> 開始する
2. ユーザーがフォーカスタイム中に画面を見る -> 残り時間と現在状態を確認する -> 終了後に休憩メッセージを認識する

## 画面 / コンポーネント範囲
- 画面:
  - フォーカスタイマー単一画面
- コンポーネント:
  - タイトルまたは用途説明
  - 時間設定入力
  - 残り時間表示
  - アナログ時計表示の残り時間ダイヤル
  - 現在状態表示
  - 開始ボタン
  - 停止ボタン
  - リセットボタン
  - 終了後メッセージ
- 状態:
  - 開始前
  - 動作中
  - 停止中
  - 終了後

## 操作ルール
- 主操作:
  - 時間を設定する
  - タイマーを開始する
  - 必要に応じて停止する
  - 初期状態へリセットする
- 入力検証とエラー時の挙動:
  - 時間設定は無効な値を受け付けない
  - 設定値が不正な場合は開始できない
  - 動作中は意図しない設定変更を防ぐ
- 空状態、読み込み中、失敗時の状態:
  - 空状態は不要
  - 読み込み中の状態は基本不要
  - 音が再生されない場合でも、終了状態と固定メッセージで区切りが分かるようにする

## 文言メモ
- ラベル:
  - 「フォーカスタイム」
  - 「時間」
  - 「開始」
  - 「停止」
  - 「リセット」
- 補助文:
  - 初期値が 25 分であることを分かるようにする
  - この画面が集中時間を確保するためのものだと短く説明する
  - 残り時間はアナログ時計で見せつつ、必要に応じて補助的な数値も併記できる
- レビュープロンプト:
  - いまどの状態か
  - あとどれくらいか
  - 次に何をすればよいか

## アクセシビリティと品質メモ
- キーボード操作とフォーカス:
  - 開始、停止、リセット、時間入力がキーボードで操作できる
- コントラストと可読性:
  - 残り時間と状態表示は遠目でも読み取りやすい
  - アナログ時計の針は背景から十分に判別できる
- レスポンシブ挙動:
  - ノート PC でもスマートフォン幅でも単一画面で情報が収まる

## 受け入れ確認
1. 初見のユーザーが、説明なしでもタイマー開始まで進める。
2. 動作中は、残り時間と現在状態がひと目で分かる。
3. 終了時は、一般的なアラーム音または画面表示によって区切りを認識できる。
