## UX 意図
業務ユーザーとレビュアーが、要求から仕様、実装レビュー、デプロイ準備までの流れを一目で理解できるようにする。

## 利用フロー
1. 業務要求を入力する -> AI が確認質問を出す -> 構造化された要件要約が見える
2. レビュアーが生成物を開く -> 関連する意図と検証メモを確認する -> 承認または修正依頼を出す

## 画面 / コンポーネント範囲
- 画面:
  - 要求受付
  - 要件明確化
  - レビューパッケージ
  - デプロイ準備サマリー
- コンポーネント:
  - 業務要求入力欄
  - 構造化要件カード
  - 追跡リンク
  - レビューチェックリスト
  - デプロイ引き継ぎサマリー
- 状態:
  - 下書き
  - レビュー中
  - デプロイ引き継ぎ準備完了

## 操作ルール
- 主操作:
  - 業務要求を送る
  - 要件を具体化する
  - 生成された実装要約を確認する
  - 承認または修正依頼を出す
- 入力検証とエラー時の挙動:
  - 要件詳細が足りない場合は確認質問を出す
  - 連動する仕様資産が足りない場合は準備完了状態にしない
- 空状態、読み込み中、失敗時の状態:
  - 空状態では流れ全体の意味を説明する
  - 失敗時の状態ではどの段階が未完了かを示す

## 文言メモ
- ラベル:
  - 技術用語より先に業務に通じるラベルを使う
- 補助文:
  - 各段階の意味、特にレビューとデプロイ準備の必要性を説明する
- レビュープロンプト:
  - 何が変わったか
  - なぜ変わったか
  - 何を検証したか
  - 何を人間がまだ確認すべきか

## アクセシビリティと品質メモ
- キーボード操作とフォーカス:
  - 主要な遷移はキーボードで操作できる
- コントラストと可読性:
  - デモ環境でも段階や状態が読み取れる
- レスポンシブ挙動:
  - ノート PC サイズでも流れを理解できる

## 受け入れ確認
1. 実装詳細を読まなくても作業フローが理解できる。
2. レビュアーが同じ画面上で提案とレビューパッケージを確認できる。
3. デプロイ引き継ぎが飛躍ではなく、境界のある準備段階として見える。
