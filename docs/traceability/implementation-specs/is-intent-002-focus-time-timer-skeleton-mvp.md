## 基盤との整合
この実装は、フォーカスタイムタイマー MVP を、フロントエンドのみで成立する単一画面アプリとして具体化する。あわせて、後続のデモで見た目や操作性をその場で改善しやすい、素朴な初期状態を意図的に作る。

## 目標
まずは「動くが味気ない」スケルトンレベルの MVP を実装する。

この段階で重要なのは、次のことが成立していることである。
- 既定値 25 分で開始できる
- 数字を書き換えて時間を変更できる
- 使用中は残り時間が見える
- 終了後にアラーム音とメッセージで終了が分かる

## スコープ
- 対象範囲:
  - 単一画面のフロントエンド実装
  - 開始前:
    - 時間を入力または変更できるフィールド
    - 開始ボタン
    - 初期値 25 分
  - 使用中:
    - 残り時間の表示
    - 現在進行中であることの状態表示
    - 停止ボタン
    - リセットボタン
  - 終了後:
    - 一般的なアラーム音
    - 「終了しました」メッセージ
    - 「休憩時間です」などの固定表示
  - 残り時間は 1 秒ごとに更新して表示すること
  - 最小限のスタイル適用
  - 実装コードは `src/focus-time-timer/` に配置する
- 対象外:
  - 実行履歴の保存
  - 音楽再生
  - 時計時刻で鳴らすタイマー
  - タイムゾーン対応
  - データベース
  - 高度な Look & Feel 作り込み
  - デジタル表示からアナログ表示への変更
  - 停止後の再開機能

## 変更契約
- 許可される変更:
  - 基本タイマー挙動を成立させるための実装
  - 残り時間表示の明確化
  - 終了時の画面表示と音の最小実装
  - スケルトンらしい最小限の整形
- 禁止される変更:
  - MVP 対象外機能を先回りで追加すること
  - 見た目を作り込みすぎて、後続デモの改善余地を減らすこと
  - バックエンドやデータベースを前提にすること
- 承認が必要:
  - 再開機能や表示方式変更など、デモ中の追加要件として見せたい項目を事前実装すること
  - UI を大きく作り込み、初期状態が味気なくなくなる変更
- 検証:
  - 25 分既定で開始できること
  - 数値変更ができること
  - 使用中に残り時間が見えること
  - 終了時にアラーム音とメッセージが出ること
- 巻き戻し:
  - スコープが膨らんだ場合は、スケルトン MVP に戻す

## 実行メモ
- 変更してよいファイル:
  - `src/focus-time-timer/**`
  - 必要最小限の設定ファイル
- 明示的に対象外のファイル:
  - `docs/**` の大幅な再構成
  - 履歴保存や外部連携のための新規 backend 資産
- リスク:
  - 見た目を整えすぎると、デモ中の改善余地が見えにくくなる
  - 音再生はブラウザ挙動に影響されうるため、終了メッセージでも区切りが分かるようにする必要がある
  - 残り時間表示は単純に 1 秒ずつ減算するのではなく、開始時刻と現在時刻から都度再計算するほうが安定する

## 実装の流れ
1. 単一画面の最小 UI を作る。
2. 25 分既定の時間入力と開始前状態を実装する。
3. 開始後に残り時間が減っていく表示を実装する。
4. 停止とリセットを実装する。
5. 終了時にアラーム音とメッセージを表示する。
6. 見た目は最低限に留め、デモ中の追加改善余地を残す。

## 検証
1. 初期表示で 25 分がセットされ、数値を書き換えられることを確認する。
2. 開始後、残り時間が 1 秒ごとに画面上で確認できることを確認する。
3. 終了時に「終了しました」メッセージと固定の休憩表示が出ることを確認する。
4. 初期状態の見た目が過度に作り込まれていないことを確認する。

## デモで後から足しやすい変更例
- 残り時間表示をデジタルからアナログ風に変える
- Look & Feel を業務向けに整える
- 停止から再開できるようにする
